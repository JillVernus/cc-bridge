# Agent Configuration Guide

This file defines AI agent specializations for different roles in cc-bridge development.

**How to use**:
```
@claude /backend
```
This switches the agent to backend specialist mode and loads the appropriate context.

## Available Modes

| Mode | Best For | Key Files |
|------|----------|-----------|
| `general` | Any task (default) | Full project context |
| `backend` | Go development | handlers, providers, config, testing |
| `frontend` | Vue 3 UI work | components, composables, styling |
| `database` | Schema/migrations | DDL/SQL, versioning |
| `release` | Versioning/deployment | VERSION, CHANGELOG, Docker |

## How Modes Work

1. **Agent loads** with minimal context
2. **User specifies mode** using `@claude /backend`
3. **Agent loads specialized guide** (e.g., `.github/instructions/backend.md`)
4. **Context filters** by applyToPatterns (only shows relevant files)
5. **Instructions tailored** to the role

## Default Behavior

Without specifying a mode, the agent uses **general mode** and loads the full instruction set from `.github/copilot-instructions.md`.
